### 任务总结：大厂 Agent 实习投递与面试准备

**结论：**
现有资料**充分支持**面试真题复盘与技术栈梳理，能够明确大厂 Agent 岗位的高频考点与面试流程；但**无法满足**“获取内推资源”这一目标，资料中仅提及内推入口存在，未提供具体的内推码或联系人。建议优先利用现有资料进行技术冲刺，同时需通过其他渠道（如职场社交平台、校友网络）补充内推资源。

**关键事实与发现：**

1.  **高频技术考点（Agent/LLM 方向）：**
    *   **核心架构：** 必须掌握 Function Calling 底层原理（Token 生成而非直接调用）、MCP（模型上下文协议）与 Skills 的区别、ACP（Agent 通信协议）设计思路。
    *   **大模型基础：** Prefix LM 与 Causal LM 区别、Attention 机制本质（Q/K/V 加权聚合）、长上下文模型选型（32K/128K+）。
    *   **工程落地：** RAG 流程（文档切分、上下文管理）、Agent 评测、安全与部署优化、ReAct 与 Plan-Execute-Replan 区别。
    *   **传统基础：** 即使投递 Agent 岗，仍会考察计算机网络（HTTP/HTTPS、TCP）、操作系统（进程/线程、内存）、数据库（MySQL 隔离级别、Redis 结构）及手撕算法（LeetCode Hot 100 及变种）。

2.  **面试流程与时间线：**
    *   **典型周期：** 以腾讯暑期实习为例，从一面到 Offer 发放约需 2 周（如 3.5 一面 - 3.18 Offer）。
    *   **面试轮次：** 通常为 3 轮技术面 + 1 轮 HR 面，部分部门包含面委笔试。
    *   **招聘动态：** 淘天集团 27 届暑期实习生正在招聘，各方向海量 HC；腾讯 PCG、IEG 等部门均有后端/算法岗开放。

3.  **备考策略与避坑指南：**
    *   **项目深度：** 避免“烂大街”项目（如黑马点评），需深入理解项目漏洞、技术方案选型及系统性思维。面试官更关注知识点如何落地实操。
    *   **算法能力：** 手撕代码是硬门槛，需熟练掌握常见数据结构（链表、栈、树）及场景题（如 IP 黑名单系统设计）。
    *   **思维模式：** 拒绝死记硬背八股文，需具备将知识点串联的系统性思维，能够应对“意想不到的角度拷打”。
    *   **辅助工具：** 可利用牛客网"AI 模拟面试”功能进行自我介绍构思与专项练习。

**资料不足与缺口：**

1.  **内推资源缺失：** 资料中多次出现“欢迎看我置顶帖子投递”、“欢迎后台联系”等话术，但**未提供具体的内推码、链接或联系人信息**，无法直接达成“获取内推资源”的任务目标。
2.  **具体部门画像模糊：** 虽提及腾讯、淘天招聘，但缺乏针对"Agent 实习”的具体部门列表（如哪些实验室或业务线核心做 Agent），可能导致投递精准度不足。
3.  **缺乏结构化冲刺计划：** 资料多为零散面经与知识点汇总，**未提供基于时间轴的复习计划表**（如第一周复习什么，第二周模拟什么），需求职者自行制定。
4.  **录取率数据空白：** 缺乏各大厂 Agent 岗位的具体竞争比例或录取偏好数据，难以量化评估录取概率。
